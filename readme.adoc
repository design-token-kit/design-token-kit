[[overview]]
== Overview

The project provides tools for working with design tokens.

Design tokens use a format where design rules are stored in a machine-readable way.
These rules can later be converted to CSS and other UI formats.

Supported input formats:

* DTCG JSON
* HRDT YAML

HRDT stands for Human-Readable Design Tokens. It is a compact YAML-based token
format designed to be easier for humans to read and write than raw DTCG JSON.

HRDT YAML files are automatically converted to DTCG JSON and then passed to the
standard command pipeline: validation, CSS conversion, and showcase generation.

Modules:

* `@design-token-kit/core` - core library.
  It provides the main functionality:
    ** design token validation
    ** design token conversion to CSS
    ** HTML showcase page generation

* `@design-token-kit/cli` - command line tool.

Node.js 18 or newer is required.

[[build]]
== Build

Install dependencies:

----
npm install
----

Build workspace packages:

----
npm run build
----

`npm run build` creates module build outputs:

* `core/lib`
* `cli/bin`

Prepare distribution directories:

----
npm run dist
----

`npm run dist` runs `build` and then creates publish-ready package directories:

* `core/build/dist`
* `cli/build/dist`

[[cli]]
== CLI

Run without local installation:

----
npx @design-token-kit/cli validate tokens.json
----

Global installation:

----
npm install -g @design-token-kit/cli
dtokens validate tokens.json
----

Run from repository sources after build:

----
npm run build
npm run dtokens -- validate core/tokens/valid.json
----

For commands with CLI options, call the built entrypoint directly:

----
node cli/bin/index.mjs convert core/tokens/valid.yaml --inform hrdt --outform css
----

[[commands]]
=== Commands

* `validate` - validates design tokens.
* `convert` - converts design tokens to CSS.
* `showcase` - builds an HTML showcase from design tokens or CSS.

[[validate]]
=== Validate

----
dtokens validate tokens.json
dtokens validate light.json dark.json
----

Run AJV schema validation only:

----
dtokens validate tokens.json
----

What the validation checks:

* schema conformance - required fields, value shapes, allowed ranges (color
  alpha, gradient stop position), non-empty gradient arrays
* broken references - alias target does not exist in the document
* reference to a group - alias points to a group instead of a token
* type mismatch - token with explicit `$type` references a token of a different type
* circular references - self-reference or reference cycle (a -> b -> a)
* broken `$extends` - group extends a non-existent path
* duplicate gradient stop positions - two stops share the same position value
* deprecated token reference - referencing a token marked `$deprecated` produces a warning

Schema errors are reported first; if any schema error is found, semantic validation is skipped.

[[convert]]
=== Convert

Print CSS to stdout:

----
dtokens convert tokens.json
----

Write CSS to file:

----
dtokens convert tokens.json --out tokens.css
----

[[showcase]]
=== Showcase

Print HTML showcase to stdout:

----
dtokens showcase tokens.json
----

Write HTML showcase to file:

----
dtokens showcase tokens.json --out showcase.html
dtokens showcase tokens.css --out showcase.html
----
